# MiniMax Tool Call Format Fix - Final Summary

## Overview
Fixed all 5 known malformed MiniMax tool call format variants generated by the `ollama-cloud/minimax-m2.5` model. Tool calls are now completely hidden from users - only final answers are shown.

## All Supported Formats

### 1. **Colon Style with Tags**
```json
{"name": "web_fetch">
<parameter name="max_chars">8000</parameter>
<parameter name="url">https://example.com</parameter>
</invoke>
</minimax:tool_call>
```
- Pattern: `{"name":`
- Has `<parameter>` tags
- Has closing `</minimax:tool_call>`

### 2. **Equals Style with Tags**
```json
{"name=web_search>
<parameter name="count">5</parameter>
<parameter name="query">DistilBERT model</parameter>
</invoke>
</minimax:tool_call>
```
- Pattern: `{"name=`
- Has `<parameter>` tags
- Has closing `</minimax:tool_call>`

### 3. **Invoke Style**
```json
{"invoke name="shell">
<parameter name="command">ls -la</parameter>
</invoke>
</minimax:tool_call>
```
- Pattern: `{"invoke name=`
- Has `<parameter>` tags
- Has closing `</minimax:tool_call>`

### 4. **Malformed JSON (Equals Sign)**
```json
{"name="web_fetch", "arguments": {"url":"https://weather.gc.ca/..."}}
```
- Pattern: `{"name=` (uses = instead of :)
- Has `"arguments":` with JSON object
- No `<parameter>` tags
- No closing tags

### 5. **Valid JSON (Colon)** ✨ NEW
```json
{"name": "web_fetch", "arguments": {"max_chars": 3000, "url": "https://en.wikipedia.org/..."}}
```
- Pattern: `{"name":` (proper JSON)
- Has `"arguments":` with JSON object
- No `<parameter>` tags
- No closing tags

## Implementation Details

### Detection (`src/agent/dispatcher.zig` lines 100-170)
- Checks for all 5 format patterns
- Detects JSON boundaries (matching braces)
- Extracts complete tool call blocks
- Logs detection for debugging

### Parsing (`src/agent/dispatcher.zig` lines 1105-1220)
- **Special Case 1**: Valid JSON `{"name": "tool", "arguments": {...}}`
- **Special Case 2**: MiniMax `{"name": "tool">` (with > marker)
- **Special Case 3**: MiniMax `{"name=tool>` (no quotes)
- **Special Case 4**: Malformed JSON `{"name="tool", "arguments": {...}}`
- Falls back to HTML-style and invoke-style patterns

### Logging
Added comprehensive logging with `std.log.scoped(.dispatcher)`:
- `debug`: Shows raw tool call content
- `info`: Shows successful parses with tool name and arguments
- `warn`: Shows parse failures
